## 어노테이션 가이드

도메인 객체에 Storage 어노테이션을 사용하면 빌드 시 MyBatis BaseMapper가 자동 생성됩니다.

### @StorageTable

클래스에 적용하여 테이블 매핑을 정의합니다.

```java
@StorageTable("orders")  // 테이블명 지정
public class Order {
    // ...
}

@StorageTable  // 테이블명 미지정 시 클래스명을 snake_case로 변환 (Order → order)
public class Order {
    // ...
}
```

### @StorageId

기본키 필드를 지정합니다.

```java
@StorageId  // autoGenerated = true (기본값), INSERT 시 자동 키 생성
private Long id;

@StorageId(autoGenerated = false)  // 직접 ID 할당
private String id;
```

### @StorageColumn

컬럼 매핑을 정의합니다.

```java
@StorageColumn(value = "order_no")  // 컬럼명 지정
private String orderNumber;

@StorageColumn  // 컬럼명 미지정 시 필드명을 snake_case로 변환
private String customerName;  // → customer_name
```

### @StorageTransient

Mapper에서 제외할 필드를 지정합니다.

```java
@StorageTransient  // SQL에서 제외됨
private String tempCalculation;
```

## 자동 코드 생성

### 입력 (도메인 객체)

```java
package com.example.domain;

@StorageTable("orders")
public class Order {

    @StorageId
    private Long id;

    @StorageColumn(value = "order_no")
    private String orderNumber;

    @StorageColumn
    private String customerName;

    private OrderStatus status;

    private BigDecimal totalAmount;

    private LocalDateTime orderedAt;

    @StorageTransient
    private String tempCalculation;

    // getter, setter...
}
```

### 출력 (자동 생성)

빌드 시 다음 파일이 자동 생성됩니다:

```
build/generated/sources/annotationProcessor/java/main/
└── com/example/domain/
    └── mapper/
        └── OrderBaseMapper.java
```

#### OrderBaseMapper.java

```java
@Mapper
public interface OrderBaseMapper {

    @Select("SELECT id, order_no AS orderNumber, customer_name AS customerName, status, total_amount AS totalAmount, ordered_at AS orderedAt FROM orders WHERE id = #{id}")
    Order findById(Long id);

    @Select("SELECT id, order_no AS orderNumber, customer_name AS customerName, status, total_amount AS totalAmount, ordered_at AS orderedAt FROM orders")
    List<Order> findAll();

    @Insert("INSERT INTO orders (order_no, customer_name, status, total_amount, ordered_at) VALUES (#{orderNumber}, #{customerName}, #{status}, #{totalAmount}, #{orderedAt})")
    @Options(useGeneratedKeys = true, keyProperty = "id", keyColumn = "id")
    int insert(Order domain);

    @Update("UPDATE orders SET order_no = #{orderNumber}, customer_name = #{customerName}, status = #{status}, total_amount = #{totalAmount}, ordered_at = #{orderedAt} WHERE id = #{id}")
    int update(Order domain);

    @Delete("DELETE FROM orders WHERE id = #{id}")
    int deleteById(Long id);

    @Select("SELECT COUNT(*) FROM orders")
    long count();

    @Select("SELECT COUNT(*) > 0 FROM orders WHERE id = #{id}")
    boolean existsById(Long id);
}
```

### 자동 생성 메서드

| 메서드 | SQL | 설명 |
|--------|-----|------|
| `findById(ID)` | `SELECT ... WHERE id = #{id}` | ID로 조회 |
| `findAll()` | `SELECT ...` | 전체 조회 |
| `insert(D)` | `INSERT INTO ...` | 삽입 (자동 키 생성) |
| `update(D)` | `UPDATE ... WHERE id = #{id}` | 수정 |
| `deleteById(ID)` | `DELETE ... WHERE id = #{id}` | ID로 삭제 |
| `count()` | `SELECT COUNT(*)` | 개수 조회 |
| `existsById(ID)` | `SELECT COUNT(*) > 0 ...` | 존재 여부 확인 |

## Mapper 확장하기

자동 생성된 BaseMapper를 상속하여 커스텀 메서드를 추가합니다.

### 방법 1: 어노테이션 방식

```java
@Mapper
public interface OrderMapper extends OrderBaseMapper {

    // 상태로 조회
    @Select("SELECT id, order_no AS orderNumber, customer_name AS customerName, status, total_amount AS totalAmount, ordered_at AS orderedAt FROM orders WHERE status = #{status}")
    List<Order> findByStatus(OrderStatus status);

    // 고객명으로 조회
    @Select("SELECT * FROM orders WHERE customer_name = #{customerName}")
    List<Order> findByCustomerName(String customerName);
}
```

### 방법 2: XML 방식

#### OrderMapper.java (인터페이스)

```java
@Mapper
public interface OrderMapper extends OrderBaseMapper {

    // 복잡한 조건 검색 (XML로 구현)
    List<Order> findByCondition(OrderSearchCondition condition);
}
```

#### OrderMapper.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.domain.mapper.OrderMapper">

    <select id="findByCondition" resultType="com.example.domain.Order">
        SELECT
            id,
            order_no AS orderNumber,
            customer_name AS customerName,
            status,
            total_amount AS totalAmount,
            ordered_at AS orderedAt
        FROM orders
        <where>
            <if test="status != null">
                AND status = #{status}
            </if>
            <if test="customerName != null and customerName != ''">
                AND customer_name LIKE CONCAT('%', #{customerName}, '%')
            </if>
            <if test="minAmount != null">
                AND total_amount &gt;= #{minAmount}
            </if>
            <if test="maxAmount != null">
                AND total_amount &lt;= #{maxAmount}
            </if>
        </where>
        ORDER BY ordered_at DESC
    </select>

</mapper>
```

### 방법 3: 혼용 (어노테이션 + XML)

```java
@Mapper
public interface OrderMapper extends OrderBaseMapper {

    // 어노테이션 방식
    @Select("SELECT * FROM orders WHERE status = #{status}")
    List<Order> findByStatus(OrderStatus status);

    // XML 방식 (어노테이션 없이 시그니처만)
    List<Order> findByCondition(OrderSearchCondition condition);
}
```

### XML 위치 설정

```yaml
# application.yml
mybatis:
  mapper-locations: classpath*:com/example/**/mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true
```

## 혼용 규칙

| 상황 | 결과 |
|------|------|
| 어노테이션만 있음 | 어노테이션 SQL 사용 |
| XML만 있음 | XML SQL 사용 |
| 둘 다 있음 | **XML이 우선** (어노테이션 무시) |

BaseMapper의 메서드를 XML로 재정의하면 자동 생성된 어노테이션 SQL 대신 XML SQL이 사용됩니다.

```xml
<!-- BaseMapper의 findById를 재정의 (조인 쿼리로 변경) -->
<select id="findById" resultType="com.example.domain.Order">
    SELECT o.*, 
           (SELECT COUNT(*) FROM order_items WHERE order_id = o.id) AS itemCount
    FROM orders o
    WHERE o.id = #{id}
</select>
```

## Adapter 사용법

### MybatisAdapter

Mapper를 감싸는 Adapter 베이스 클래스입니다.

```java
@Adapter
public class OrderMybatisAdapter extends MybatisAdapter<Order, Long, OrderMapper>
        implements OrderPort {

    public OrderMybatisAdapter(OrderMapper mapper) {
        super(Order.class, mapper);
    }

    // BaseMapper 메서드 위임 (필수 구현)
    @Override
    protected Order doFindById(Long id) {
        return mapper.findById(id);
    }

    @Override
    protected List<Order> doFindAll() {
        return mapper.findAll();
    }

    @Override
    protected int doInsert(Order domain) {
        return mapper.insert(domain);
    }

    @Override
    protected int doUpdate(Order domain) {
        return mapper.update(domain);
    }

    @Override
    protected int doDeleteById(Long id) {
        return mapper.deleteById(id);
    }

    @Override
    protected long doCount() {
        return mapper.count();
    }

    @Override
    protected boolean doExistsById(Long id) {
        return mapper.existsById(id);
    }

    // ==================== Port 인터페이스 구현 ====================

    @Override
    public Order save(Order order) {
        if (order.getId() == null) {
            return insertWithException(order);
        } else {
            return updateWithException(order);
        }
    }

    @Override
    public Optional<Order> findById(Long id) {
        return findByIdWithException(id);
    }

    @Override
    public List<Order> findByStatus(OrderStatus status) {
        return mapper.findByStatus(status);  // 커스텀 메서드
    }

    @Override
    public List<Order> search(OrderSearchCondition condition) {
        return mapper.findByCondition(condition);  // XML 메서드
    }
}
```

### MybatisAdapter 제공 메서드

| 메서드 | 예외 변환 | 설명 |
|--------|----------|------|
| `findById(ID)` | ❌ | ID로 조회 (Optional) |
| `findByIdWithException(ID)` | ✅ | ID로 조회 (예외 변환) |
| `findByIdOrThrow(ID)` | ✅ | ID로 조회, 없으면 예외 |
| `findAll()` | ❌ | 전체 조회 |
| `findAllWithException()` | ✅ | 전체 조회 (예외 변환) |
| `insert(D)` | ❌ | 삽입 |
| `insertWithException(D)` | ✅ | 삽입 (예외 변환) |
| `update(D)` | ❌ | 수정 |
| `updateWithException(D)` | ✅ | 수정 (예외 변환) |
| `deleteById(ID)` | ❌ | 삭제 |
| `deleteByIdWithException(ID)` | ✅ | 삭제 (예외 변환) |
| `count()` | ❌ | 개수 조회 |
| `existsById(ID)` | ❌ | 존재 여부 확인 |

## 예외 처리

### StorageException

영속성 계층 예외를 도메인 예외로 변환합니다.

```java
// 정적 팩토리 메서드
StorageException.of("저장 실패");
StorageException.of("저장 실패", cause);
StorageException.saveFailed("Order", cause);
StorageException.notFound("Order", 1L);
StorageException.deleteFailed("Order", cause);
StorageException.duplicate("Order", "orderNumber", "ORD-001");
```

### 에러 코드

| 코드 | 설명 |
|------|------|
| `STORAGE_SAVE_FAILED` | 저장 실패 |
| `STORAGE_NOT_FOUND` | 조회 실패 (데이터 없음) |
| `STORAGE_DELETE_FAILED` | 삭제 실패 |
| `STORAGE_DUPLICATE` | 중복 데이터 |
| `STORAGE_ERROR` | 일반 오류 |

## 어노테이션 변환 규칙

| Storage 어노테이션 | SQL 변환 | 비고 |
|-------------------|---------|------|
| `@StorageTable` | 테이블명 | 미지정 시 snake_case 변환 |
| `@StorageId` | WHERE id = #{id} | autoGenerated=true 시 useGeneratedKeys |
| `@StorageColumn` | 컬럼 AS 필드 | SELECT에서 alias 사용 |
| `@StorageTransient` | (SQL 제외) | 모든 SQL에서 제외 |

## 주의사항

1. **도메인 객체에 getter/setter 필요**: MyBatis가 getter/setter를 사용하여 값을 매핑합니다.

2. **Enum 타입 처리**: MyBatis는 기본적으로 Enum의 name()을 사용합니다. 커스터마이징이 필요하면 TypeHandler를 등록하세요.

3. **XML 파일 위치**: `mapper-locations` 설정을 확인하세요.
   ```yaml
   mybatis:
     mapper-locations: classpath*:com/example/**/mapper/*.xml
   ```

4. **카멜케이스 ↔ 스네이크케이스**: 자동 변환을 위해 설정을 추가하세요.
   ```yaml
   mybatis:
     configuration:
       map-underscore-to-camel-case: true
   ```

5. **BaseMapper vs OrderMapper**: Bean 주입 시 사용자가 만든 `OrderMapper`를 주입받으세요. BaseMapper는 상속용입니다.

## JPA vs MyBatis 비교

| 항목 | demo-storage-jpa | demo-storage-mybatis |
|------|-----------------|---------------------|
| 자동 생성 | Entity, Mapper, Repository | BaseMapper |
| SQL | 자동 생성 (JPA) | 어노테이션에 직접 작성 |
| 복잡한 쿼리 | QueryDSL | XML 동적쿼리 |
| 확장 방식 | Repository 상속 | Mapper 상속 |
| 도메인 객체 | Entity 별도 생성 | 도메인 객체 직접 사용 |
